# Retail-Sales-Inventory-Intelligence-System

## Project Overview
This project focused on building an end-to-end Retail Sales & Inventory Intelligence System using Python, PostgreSQL and Power BI. The objective was to analyze retail business operations including sales, inventory, staff performance, and regional sales distribution.

## Technologies Used
- Python
- Pandas
- PostgreSQL
- SQL
- Power BI

## Project Workflow
•	Imported raw retail datasets into Python using Pandas.

•	Performed data cleaning including null handling, datatype conversion, duplicate validation, and normalization.

•	Integrated and loaded cleaned datasets into PostgreSQL.

•	Created relational database structure using primary and foreign key relationships.

•	Developed SQL queries and views for business analysis.

•	Connected PostgreSQL to Power BI.

•	Built interactive dashboards with KPIs, charts, slicers, maps, and matrix tables.

## Data Cleaning & Pre-processing
•	Handled null values in phone numbers, shipped dates, and manager IDs.

•	Removed duplicate records and validated primary key uniqueness.

•	Converted date columns into proper datetime format.

•	Standardized product and category names.

•	Validated relational join keys across datasets.

## SQL Analytics & Insights
•	Store-wise and Region-wise Sales Analysis

•	Product-wise Sales and Inventory Trends

•	Customer Order Frequency Analysis

•	Revenue and Discount Analysis

•	Staff Performance Reports

•	Shipment Delay Analysis

•	Reusable SQL Views for Dashboard Reporting

## Power BI Dashboard Features
•	KPI Cards: Total Sales, Orders, Customers, Discounts

•	Sales Trend Line Charts

•	Regional Sales Distribution Maps

•	Category and Store Performance Bar Charts

•	Inventory Analysis Tables and Matrix Views

•	Interactive Slicers for Date, Region, Store, Category, and Brand
